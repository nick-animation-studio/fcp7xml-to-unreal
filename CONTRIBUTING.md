# Contributing Guidelines

:tada: **First off, thank you for considering contributing to our project!** :tada:

This is a community-driven project, so it's people like you that make it useful and
successful. These are some of the many ways to contribute:

* :bug: Submitting bug reports and feature requests
* :memo: Writing tutorials or examples
* :mag: Fixing typos and improving the documentation
* :bulb: Writing code for everyone to use
* :people_holding_hands: Community engagement and outreach (post about this, star the project on GitHub)

If you get stuck at any point you can create an
[issue](https://github.com/nick-animation-studio/fcp7xml-to-unreal/issues) on GitHub.

For more information on contributing to open source projects,
[GitHub's own guide](https://opensource.guide/how-to-contribute)
is a great starting point if you are new to version control.

## Ground Rules

The goal is to maintain a diverse community that's pleasant for everyone.
**Please be considerate and respectful of others**. Everyone must abide by our
[Code of Conduct](./CODE_OF_CONDUCT.md)
and we encourage all to read it carefully.

When contributing to this repository, please first discuss the change you wish to make via issue, email, or any other method with the owners of this repository before making a change.

## Pull Requests

We actively welcome your pull requests.

If you're new, we encourage you to take a look for any issues tagged with [good first issue](https://github.com/nick-animation-studio/fcp7xml-to-unreal/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22).

### Making Code Contributions

1. Fork this project on GitHub, and make a branch in your fork from `main` - this will be the home for your proposed changes.
2. While developing, enable the git `pre-commit` checks that are specified for the project. This will ensure your code is formatted and linted appropriately.
3. Follow [PEP-8](https://peps.python.org/pep-0008/) for style questions that aren't covered by the `pre-commit` checks.
4. Write tests as appropriate for your changes, and make sure all existing tests pass.
5. Commit your changes with a commit message - please follow [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
6. Push your changes to GitHub.
7. Submit a Pull Request.
